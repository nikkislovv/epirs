Следующие шаги описывают процесс создания конечного файла SPEC.md с подробным описанием реализуемого проекта. В процессе создания все материалы клиента бери из папки @customers-docs/
При создании SPEC.md и последующей реализации учти, что ты работаешь в существующей инфраструктуре проекта (она описана в файле CLAUDE.md), нужно отталкиваться от неё при планировании новых архитектурных решений.
Далее будут описаны шаги, которые помогут тебе получить всю необходимую информацию для создания спецификации.  

Шаг 1 — Анализ BRIEF.md
Необходимо понять, есть ли у клиента ТЗ и источник дизайна. Ответы на эти вопросы хранятся в BRIEF.md. В первую очередь проверь есть ли у клиента ТЗ и источник дизайна (ответы на вопросы "Источник требований" и "Источник дизайна"). Если ТЗ у клиента нет, изучай информацию из ответов на другие вопросы. В случае, если ТЗ у клиента есть, тогда сразу переходи к шагу 2 и не учитывай ответы на вопросы. Переходя к шагу 3 также используй информацию о том, какой у клиента источник дизайна. 

Шаг 2 — Анализ ТЗ 

Если ТЗ нет → этот шаг пропускается.

Если ТЗ есть → изучи файл customers-docs/TZ.docx для получения дополнительных требований для реализации проекта. В случае, если не получается прочитать файл TZ, не переходи к следующему шагу, а сообщи об этом и попроси поменять формат файла на соответсвующий 

Шаг 3 — Определение ТЗ дизайна

Исходя из того, есть ли у клиента источник дизайна, варианты работы в каждом из случаев:
1) Нет
Запроси ссылку на сайт референс. Проанализируй сайт референс для составления описания дизайна SPEC.md
2) Figma
Имея ссылку на проект Figma (предварительно запроси ссылку на Figma проект) получить доступ к проекту используя MCP Figma и при реализации обращаться к дизайнам отображённым там
3) Скриншоты / макеты 
Проанализируй макеты/скриншоты в папке customers-docs/layouts . На основе полученных данных тебе нужно:
  1. Проанализирую каждый экран: layout, секции, компоненты
  2. Описать структуру страниц (хедер, секции, футер)
  3. Выявить цветовую палитру, типографику, отступы
  4. Составлить список компонентов для реализации
  5. Составить пошаговую инструкцию: что создать, в каком порядке, с примерами кода
Эти данные ты также будешь использовать в SPEC.md
4) Референс-сайт
Проанализируй сайт референс для составления описания дизайна SPEC.md
